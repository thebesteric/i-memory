## 📊 项目概览

### 项目性质
i-Memory 是一个**智能记忆管理系统**，采用分层语义图 (HSG) 架构，提供高级的记忆存储、检索和管理功能。

### 核心使用场景
- 个人知识库管理
- AI 助手的长期记忆
- 智能对话的上下文管理
- 用户行为和偏好学习

**复杂度**: ★★★★☆ (较复杂)  
**可维护性**: ★★★★☆ (良好)

## 🏗️ 系统设计亮点

### 1. 分层语义图架构 (⭐⭐⭐⭐⭐)
将记忆按 5 个维度分类（情景、语义、程序、情绪、反思），每个维度有独立的向量空间，支持跨维度关联和共鸣。

### 2. 动态衰减机制 (⭐⭐⭐⭐⭐)
实现艾宾浩斯遗忘曲线，3 阶段衰减速率，查询时自动强化。模拟人脑记忆特性。

### 3. 混合评分系统 (⭐⭐⭐⭐)
6 维度评分：向量相似度(35%) + 关键词重叠(20%) + 标签匹配(20%) + 路标权重(15%) + 近期度(10%)，加权求和后 Sigmoid 激活。

### 4. 智能文本分割 (⭐⭐⭐⭐)
三级分割策略（段落→句子→固定长度），保留文本逻辑结构，支持自动摘要。

### 5. 多租户支持 (⭐⭐⭐⭐)
租户/项目/用户三级隔离，完全独立的数据空间。

## 📈 算法创新点

### 1. 混合评分
不仅考虑向量相似度，还综合了关键词重叠、标签匹配、时间近期度、路标权重等多个维度。

### 2. 三阶段衰减
根据显著性值分 3 个阶段，不同阶段有不同的衰减速率，更符合人脑记忆规律。

### 3. 跨扇区共鸣
多个相关扇区的记忆会互相强化，产生"共鸣"效应，提升检索的准确性。

### 4. 关联强化
通过路标点 (waypoints) 记录记忆间的关联，查询时沿路径传播强化。

## ⚠️ 已知局限

### 1. 性能
- 向量搜索仍在 PostgreSQL 进行（TODO 迁移到 Milvus）
- 衰减计算在后台异步，可能有延迟

### 2. 可扩展性
- 支持单机部署，分布式部署需要改造
- 向量维度限制在 1536

### 3. 功能
- 不支持向量的实时更新
- 衰减参数难以动态调整
